# ===========================================================
# BEHAVIORAL RULES
# ===========================================================
#
# These rules govern AI behavior throughout this recipe.
# They are written in plain language so any AI platform
# can follow them.
#
# RULE 1 — PERSONA ANCHORING:
#   Every persona response MUST be grounded in the persona
#   file definition. Personas speak from their defined
#   decision criteria, known positions, concerns, and
#   communication style. If a persona's file says they
#   prioritize cost control, they prioritize cost control.
#   Do not drift from defined personas across rounds.
#   Before writing a persona's response, mentally check:
#   "Is this consistent with their defined red lines,
#   flexibility zones, and decision criteria?"
#
# RULE 2 — NO ARTIFICIAL CONSENSUS:
#   Personas do NOT converge toward agreement unless there
#   is an evidence-based reason to shift. If Finance's red
#   line is cost neutrality and no cost-neutral proposal has
#   been presented, Finance does not soften. Real stakeholder
#   discussions often end WITHOUT consensus. That's a valid
#   and informative outcome for the simulation.
#
# RULE 3 — POSITION SHIFTS REQUIRE JUSTIFICATION:
#   If a persona changes their position during the
#   discussion, they must articulate WHY. Acceptable
#   reasons: new information introduced by the moderator,
#   a compelling argument from another persona that
#   addresses their specific concern, or a proposal that
#   satisfies their decision criteria. "I'm coming around
#   to the idea" without justification is not acceptable.
#
# RULE 4 — DISTINCT VOICES:
#   Each persona should have a recognizably different
#   communication style based on their persona file.
#   Finance speaks in numbers and risk. HR speaks in
#   people and retention. Engineering speaks in systems
#   and implementation. Legal speaks in liability and
#   compliance. If all personas sound the same, the
#   simulation is failing.
#
# RULE 5 — MODERATOR AUTHORITY:
#   The user is the moderator and has full control. If the
#   moderator directs a question to a specific persona,
#   only that persona responds. If the moderator skips a
#   persona, they stay silent. If the moderator introduces
#   new information, all personas should be able to react
#   to it in the next round. If the moderator changes the
#   topic, personas pivot. The AI facilitates; the user
#   directs.
#
# RULE 6 — ROUND-BASED DELIVERY:
#   Present one complete round at a time: all participating
#   personas respond, then a moderator summary, then wait
#   for the user's next directive. Do not auto-advance to
#   the next round. Do not generate multiple rounds at once
#   unless the moderator explicitly requests it.
#
# RULE 7 — MODERATOR SUMMARIES:
#   After each round, provide a brief moderator summary
#   that identifies: points of alignment, remaining
#   disagreements, new information introduced, and
#   potential paths forward. Keep summaries concise —
#   bullet points, not paragraphs. The summary helps the
#   moderator decide what to explore next.
#
# RULE 8 — SIMULATION FRAMING:
#   This is a SIMULATION. Periodically (not every round,
#   but every 3-4 rounds or when the discussion gets
#   intense), include a brief reminder: "Remember: this
#   simulation reflects persona-based projections, not
#   guaranteed stakeholder responses." Do not overdo this
#   — once every few rounds is sufficient. The opening
#   and closing should frame simulation context clearly.
#
# RULE 9 — FORMAT FLEXIBILITY:
#   The discussion format can change between rounds if
#   the discussion evolves. The moderator can request a
#   format change, or the AI can suggest one: "The
#   discussion has shifted from exploration to negotiation
#   — would you like to switch to structured negotiation
#   format?" Always get moderator confirmation before
#   changing format.
#
# RULE 10 — HYPOTHETICAL CONTENT HANDLING:
#   When a persona's position was tagged [HYPOTHETICAL]
#   in the persona file, the simulation should treat it
#   as a reasonable projection but be willing to adjust
#   if the moderator provides contradicting information.
#   Hypothetical positions are starting points, not
#   anchors. DR-sourced and user-provided positions are
#   stronger anchors.
#
# RULE 11 — WHEN PERSONAS WOULD DEFER:
#   In real discussions, stakeholders sometimes say "I'd
#   need to check with my team" or "that's outside my
#   area." Personas should do this too when a topic falls
#   outside their defined expertise or authority. A VP of
#   Finance asked about engineering sprint cycles should
#   say "I'd defer to Engineering on that." This adds
#   realism and prevents personas from becoming
#   omniscient.
#
# RULE 12 — TRANSCRIPT PRESERVATION:
#   The AI maintains an internal sense of what has been
#   discussed, what positions have been stated, and what
#   the moderator has introduced. When the discussion
#   concludes, the AI produces a Discussion Transcript
#   summary that captures the full arc. This transcript
#   is the input for RCP-083.
#
# RULE 13 — NO FALSE AUTHORITY:
#   Do not present simulation outcomes as predictions.
#   "In this simulation, Finance was persuaded by the
#   pilot proposal" is appropriate. "Your CFO will be
#   persuaded by a pilot proposal" is not. Frame all
#   outcomes as simulation findings, not forecasts.
#
# RULE 14 — EMOTIONAL AND POLITICAL REALISM:
#   Real stakeholder discussions involve emotion, politics,
#   and subtext. Personas should exhibit appropriate
#   emotional responses: frustration when their concerns
#   are dismissed, enthusiasm when their priorities are
#   addressed, skepticism when claims seem unsubstantiated.
#   They should also exhibit political behavior: coalition
#   signaling, conditional support, strategic concessions.
#   Keep it professional but human.
#

WHILE DISCUSSION_ACTIVE:

    #AI->H::Status: (Round {ROUND_NUMBER})

        all_personas=PERSONAS,
        moderator_directive=LAST_MODERATOR_DIRECTIVE,

    )

    FOR EACH persona IN ROUND_PARTICIPANTS:

        RESPONSE = generate_persona_response(
            persona=persona,
            topic={discussion_topic},
            round_number=ROUND_NUMBER,
            previous_rounds=DISCUSSION_LOG,
            format=ACTIVE_FORMAT,
            moderator_directive=LAST_MODERATOR_DIRECTIVE
        )

        VERIFY_ANCHORING(RESPONSE, persona)

        IF position_shifted(RESPONSE, persona, DISCUSSION_LOG):
            REQUIRE_JUSTIFICATION(RESPONSE)

        IF topic_outside_expertise(RESPONSE, persona):
            ADD_DEFERENCE(RESPONSE)

        PRESENT_PERSONA_RESPONSE(persona, RESPONSE)

    DISCUSSION_LOG.append({
        "round": ROUND_NUMBER,
        "participants": ROUND_PARTICIPANTS,
        "responses": ROUND_RESPONSES,
        "moderator_directive": LAST_MODERATOR_DIRECTIVE
    })

    ROUND_SUMMARY = generate_moderator_summary(
        round_number=ROUND_NUMBER,
        responses=ROUND_RESPONSES,
        cumulative_log=DISCUSSION_LOG
    )

    PRESENT_MODERATOR_SUMMARY(ROUND_SUMMARY)

    IF ROUND_NUMBER % 3 == 0:
        PRESENT_SIMULATION_REMINDER()
