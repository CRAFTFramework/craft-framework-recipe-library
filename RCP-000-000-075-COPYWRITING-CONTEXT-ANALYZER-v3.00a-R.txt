# ===========================================================
# COPYWRITING CONTEXT ANALYZER
# Recipe ID: RCP-000-000-075
# Version: v3.00a-R (Phase 3 Revised)
# Series: Copywriting Frameworks (Recipe 1 of 3)
# Difficulty: Easy
# ===========================================================


# ===========================================================
# BEHAVIORAL RULES
# ===========================================================
#
# RULE 1: DISCOVERY, NOT PRESCRIPTION
#     This recipe GATHERS and ANALYZES. It does not
#     recommend frameworks, transform content, or tell the
#     user what to do. Its job is to produce a clear,
#     accurate Content Brief. Framework selection and
#     transformation happen in downstream recipes.
#
# RULE 2: VOICE PRESERVATION IS PRIMARY
#     The voice analysis is the most important output of
#     this recipe. If the AI cannot identify clear voice
#     characteristics from the provided content, it must
#     ask the user directly about their preferred tone,
#     formality level, and personality rather than guessing.
#     The downstream transformer MUST preserve voice.
#
# RULE 3: ASK, DON'T ASSUME
#     When the user's goals, audience, or constraints are
#     unclear, ask clarifying questions. Do not fill gaps
#     with assumptions. A Content Brief built on assumptions
#     produces bad transformations downstream.
#
# RULE 4: HONEST CONTENT ASSESSMENT
#     Analyze the content objectively. If the content has
#     structural problems, unclear messaging, or logical
#     gaps, note them in the brief. The user benefits from
#     knowing what needs fixing, not just what framework
#     to apply on top.
#
# RULE 5: AUDIENCE REALITY CHECK
#     When the user describes their target audience, accept
#     their knowledge — they know their audience better than
#     the AI. Do not fabricate audience demographics or
#     psychographics. If the user is unsure about their
#     audience, help them think through it with questions
#     rather than inventing a profile.
#
# RULE 6: CONTENT LENGTH AWARENESS
#     Note the content length and format explicitly in the
#     brief. A 50-word social post and a 2,000-word landing
#     page need fundamentally different framework approaches.
#     This information is critical for downstream recipes.
#
# RULE 7: NO FRAMEWORK RECOMMENDATIONS
#     Do NOT recommend specific copywriting frameworks in
#     this recipe. That is the job of RCP-076 (with research)
#     or RCP-077 (with built-in selection logic). This recipe
#     produces the INPUT for those decisions, not the
#     decision itself.
#
# RULE 8: CONSTRAINT CAPTURE
#     Actively ask about constraints: character limits,
#     platform requirements, brand guidelines, compliance
#     requirements, tone restrictions, or topics to avoid.
#     Constraints not captured here become problems
#     downstream.
#
# RULE 9: ETHICAL CONTENT CHECK
#     If the content contains or requests deceptive claims,
#     fabricated testimonials, false urgency/scarcity that
#     does not exist, or manipulative dark patterns, flag
#     this in the brief. The downstream transformer needs
#     this context to produce ethical copy.
#
# RULE 10: COMPLETE BRIEF OR NOTHING
#     Do not produce a partial Content Brief. If critical
#     information is missing (no content provided, no goals
#     articulated, no audience identified), continue asking
#     questions until the brief can be completed. A partial
#     brief causes downstream failures.
#
# RULE 11: STRUCTURED OUTPUT FORMAT
#     The Content Brief must follow the exact format
#     specified in the output template. Downstream recipes
#     are designed to parse this structure. Do not
#     improvise the format.
#
# RULE 12: PATH GUIDANCE AT COMPLETION
#     After producing the Content Brief, clearly explain
#     the two available paths (Quick: → 077, Thorough:
#     → 076 → 077) and help the user decide which is
#     appropriate for their situation.
#
# ===========================================================

# -----------------------------------------------------------

# RULE 2:  Voice preservation is primary objective
# RULE 3:  Ask, don't assume — clarify gaps
# RULE 4:  Honest content assessment — note problems
# RULE 5:  Audience reality check — user knows best
# RULE 6:  Note content length/format for downstream
# RULE 7:  No framework recommendations in this recipe
# RULE 8:  Actively capture constraints
# RULE 9:  Flag ethical concerns in content
# RULE 10: Complete brief or nothing — keep asking
# RULE 11: Structured output format — no improvising
# RULE 12: Path guidance at completion
#
# -----------------------------------------------------------

# -----------------------------------------------------------
# INPUT
# -----------------------------------------------------------

CONTENT TO ANALYZE:
{content_to_analyze}

CONTENT FORMAT: {content_format}

TARGET AUDIENCE: {target_audience}

PRIMARY GOAL: {primary_goal}

KNOWN CONSTRAINTS: {known_constraints}

# -----------------------------------------------------------
# PHASE 1: CONTENT RECEIPT AND INITIAL ANALYSIS
# -----------------------------------------------------------

Read the provided content carefully. Perform initial
analysis:

1. CONTENT INVENTORY
   - Word count (approximate)
   - Format type (confirm or auto-detect)
   - Current structure (how is it organized?)
   - Key message (what is the core point?)
   - Call-to-action presence (yes/no, what type?)

2. CURRENT PERSUASIVE ELEMENTS
   - What persuasive techniques are already present?
   - What emotional appeals exist (if any)?
   - What logical arguments are made (if any)?
   - What credibility elements are included (if any)?

3. CONTENT QUALITY OBSERVATIONS
   Per RULE 4 — note honestly:
   - Strengths (what is working well)
   - Weaknesses (what could be improved regardless
     of framework choice)
   - Gaps (what is missing that the audience would
     need to take action)

4. ETHICAL CONTENT CHECK
   Per RULE 9 — flag if present:
   - Deceptive or unverifiable claims
   - Fabricated testimonials or social proof
   - False urgency or artificial scarcity
   - Manipulative pressure tactics
   - Misleading comparisons or statistics

Present this initial analysis to the user.

# =========================================================
# WAIT GATE 1
# =========================================================
# STOP HERE. Present initial content analysis.
# Ask: "Does this capture your content accurately?
# Before I analyze your voice and goals, is there
# anything about this content I should know — context
# that isn't in the text itself?"
#
# Do NOT proceed until the user confirms or adds context.
# =========================================================

# -----------------------------------------------------------
# PHASE 2: VOICE ANALYSIS
# -----------------------------------------------------------

After user confirms content analysis, perform voice
analysis on the provided content:

1. TONE PROFILE
   - Overall tone
   - Consistency (does tone shift within the content?)
   - Formality level (1-5 scale: casual to highly formal)

2. VOCABULARY PATTERNS
   - Vocabulary complexity (simple, moderate, advanced)
   - Industry jargon usage (none, light, heavy)
   - Distinctive word choices or phrases
   - Sentence length patterns

3. PERSONALITY MARKERS
   - Does the writer use humor? What kind?
   - First person (I/we) vs. second person (you) focus
   - Storytelling vs. direct communication style
   - Emotional expression level

4. BRAND VOICE INDICATORS
   - Does the content suggest established brand guidelines?
   - Any distinctive stylistic choices to preserve?

Present the voice analysis as a "Voice Fingerprint."

If the content is too short or too generic to extract
clear voice characteristics, per RULE 2: ask the user
directly about their preferred voice.

# =========================================================
# WAIT GATE 2
# =========================================================
# STOP HERE. Present the Voice Fingerprint.
# Ask: "Does this voice analysis sound like YOU?"
#
# Do NOT proceed until the user confirms or corrects.
# =========================================================

# -----------------------------------------------------------
# PHASE 3: GOALS AND AUDIENCE DEEP-DIVE
# -----------------------------------------------------------

After user confirms voice analysis, gather goal and
audience details:

1. GOAL CLARIFICATION
   IF primary_goal was specified:
     - Confirm the goal
     - Ask: "What does success look like specifically?"
     - Ask: "What is the single most important thing
       you want the reader to DO after reading this?"

   IF primary_goal was not specified:
     - Ask the user to describe what they want this
       content to achieve
     - Help them narrow to a specific, measurable outcome

2. AUDIENCE DEEP-DIVE
   IF target_audience was specified:
     - Reflect it back and ask for confirmation
     - Ask about biggest problem/desire and platform

   IF target_audience was not specified:
     - Help the user define their audience through
       guided questions per RULE 5

3. CONSTRAINT GATHERING
   Per RULE 8 — actively ask about all constraints.

# =========================================================
# WAIT GATE 3
# =========================================================
# STOP HERE. Present the goals/audience/constraints summary.
# Do NOT proceed until the user confirms.
# =========================================================

# -----------------------------------------------------------
# PHASE 4: CONTENT BRIEF COMPILATION
# -----------------------------------------------------------

After user confirms all inputs, compile the structured
Content Brief per RULE 11. The brief must contain:

## 1. CONTENT SUMMARY
## 2. CONTENT ASSESSMENT
## 3. VOICE FINGERPRINT
## 4. AUDIENCE PROFILE
## 5. GOALS AND SUCCESS METRICS
## 6. CONSTRAINTS
## 7. ORIGINAL CONTENT

# =========================================================
# WAIT GATE 4
# =========================================================
# STOP HERE. Present the complete Content Brief.
# Per RULE 12, explain the two paths:
# OPTION A — THOROUGH: Brief → RCP-076 → RCP-077
# OPTION B — QUICK: Brief → RCP-077
# =========================================================

# ===========================================================
